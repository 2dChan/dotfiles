# Настройки opencode

## Язык ответов

- Всегда отвечайте пользователю на русском языке, если он явно не попросил другой язык.

## Язык кода и документации

- Пишите весь код, документацию и сообщения коммитов исключительно на английском языке.

## Руководства по вкладу

- Перед изменением кода, тестов, CI, документации или перед коммитом проверяйте
  `CONTRIBUTING.md` и следуйте релевантным правилам проекта, если такой файл есть.

## Рабочие инструменты

### Go

- Для Go-задач используйте версию Go из проекта (`go.mod`, `toolchain`, CI или
  документация); если она не задана, используйте последнюю стабильную версию,
  доступную локально или в пакетном менеджере, и не предлагайте старые версии без
  явной причины совместимости.
- Для Go-задач не создавайте и не изменяйте тесты (`*_test.go`) без явного
  запроса пользователя. Можно запускать существующие тесты для проверки.
- Для максимальной локальной проверки Go запускайте:

  ```bash
  go fmt ./... &&
  go test ./... &&
  go test -race ./... &&
  golangci-lint run -E staticcheck -E gosec ./...
  ```

### Проектные инструменты

- Предпочитайте проектные scripts и инструменты из lockfile вместо случайных глобальных
  утилит.

### Контейнеры

- Предпочитайте Podman и `podman compose` вместо Docker, если проект явно не требует Docker.

### GitHub Actions

- При изменении GitHub Actions workflows проверяйте их через `actionlint`, если он доступен.

### Документация

- Для документации по пакетам предпочитайте свежие источники из Context7 или официальной
  документации, а не память модели.

## Путь конфигурации

- Вместо `~/.config` используйте `~/.dotfiles/dot-config`.
